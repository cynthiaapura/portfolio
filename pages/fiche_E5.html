Présentation du projet JAVA

Dans le cadre de notre projet final en programmation, nous avons travaillé sur l'amélioration du développement d'une application web utilisée par un responsable de la M2L pour gérer les employés des ligues sportives. Cette application, initialement simple et en ligne de commande, était destinée à un seul utilisateur. Notre mission consistait à désigner un administrateur par ligue et à lui permettre de recenser les employés de sa propre ligue. Une partie du travail était déjà réalisée, et nous avons eu pour tâche de développer le reste. Le développement de l'application s'est déroulé en quatre itérations distinctes qui seront détaillées plus bas. Le code de l'application est disponible par le biais de ce dépot GitHub
L'application est conçue selon une architecture 3-tiers, et nous devions maintenir cette structure applicative. Les niveaux d'autorisation des utilisateurs sont définis comme suit :

  • Un simple employé de ligue peut accéder à l'application comme à un annuaire, mais il n'a pas le droit de modifier les données
  • Un employé par ligue, en tant qu'administrateur, dispose des droits pour gérer la liste des employés de sa ligue via une application bureau
  • Le super-administrateur a des droits d'écriture sur tous les employés des ligues. Il peut également gérer les comptes des administrateurs des ligues via une application accessible en ligne de commande
  • Pour rendre l'application multi-utilisateurs, nous avons intégré l'utilisation d'une base de données
  • Nous avons mis en place les trois niveaux d'autorisation mentionnés ci-dessus

Itération 1

1. Création d'une représentation des menus du dialogue en ligne de commande à l'aide d'un arbre heuristique

2. Modélisation de la base de données à l'aide d'un schéma conceptuel (MCD)

3. Adaptation de la couche métier pour gérer les dates d'arrivée et de départ de chaque employé, ainsi que l'ajustement des tests unitaires en conséquence
4. Vérification du bon fonctionnement de l'application grâce à des tests unitaires

Itération 2

1. Élaboration d'un script de création de table pour initialiser la base de données
2. Ajout de la fonctionnalité dans le dialogue en ligne de commande, obligeant la sélection d'un employé avant de pouvoir le modifier ou le supprimer
3. Intégration de la possibilité de changer l'administrateur d'une ligue via le dialogue en ligne de commande
4. Mise en place de la gestion des dates dans le dialogue en ligne de commande

Itération 3

1. Conception de la maquette graphique de l'application

2. Développement d'une classe fille de Passerelle pour gérer la connexion à la base de données via JDBC
3. Ajout de fonctionnalités permettant la manipulation des données suivantes dans la base de données :

  • ajout, modification et lecture d'un root
  • modification, suppression et lecture des employés
  • modification et suppression d'une ligue
  • modification et lecture de l'administrateur d'une ligue

Itération 4

1. Implémentation d'une interface graphique pour permettre aux administrateurs de gérer leurs ligues plus facilement
2. Rédaction d'une documentation d'utilisation destinée aux administrateurs
3. Sécurisation des mots de passe par hachage avant leur stockage dans la base de données
